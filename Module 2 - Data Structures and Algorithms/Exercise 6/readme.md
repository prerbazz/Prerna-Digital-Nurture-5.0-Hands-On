# Library Management System

## 1. Explain linear search and binary search algorithms.

**Linear Search:** Linear search checks each element one by one until the required item is found or the list ends. It can be used on both sorted and unsorted data.

**Binary Search:** Binary search repeatedly divides the search space into two halves until the required element is found. It works only on sorted data.

## 2. Compare the time complexity of linear search and binary search.

Linear Search:
- Best Case: O(1)
- Average Case: O(n)
- Worst Case: O(n)

Binary Search:
- Best Case: O(1)
- Average Case: O(log n)
- Worst Case: O(log n)

## 3. When should each algorithm be used?

Linear search is suitable for small or unsorted datasets where sorting is not practical.

Binary search is suitable for large datasets that are already sorted because it finds elements much faster than linear search.
